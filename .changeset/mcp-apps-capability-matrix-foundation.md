### `@mcpjam/inspector`
- **MCP Apps spec-bridge per-dimension capability matrix (foundation)**: introduces `McpAppsCapabilities` / `ResolvedMcpAppsCapabilities` types and `MCP_APPS_FULL_SURFACE` / `MCP_APPS_COPILOT_SURFACE` presets that mirror Microsoft 365 Copilot's published Component-bridge table. Each host style now declares its SEP-1865 `HostCapabilities` advertisement via a 14-dimension matrix (`availableDisplayModes`, `toolInputPartial`, `toolCancelled`, `hostContextChanged`, `resourceTeardown`, `toolInfo`, `serverResources`, `logging`, `updateModelContext`, `message`, `sandboxPermissions`, `cspFrameDomains`, `cspBaseUriDomains`, `resourcePrefersBorder`) instead of a flat `hostCapabilities` blob — the new `buildHostCapabilities(matrix, augment?)` is the single derivation point. Independent from the existing `window.openai` shim matrix: `app.callTool` and `window.openai.callTool` are different surfaces and never cross-gate.
- **Per-host preset matrices wired**: Claude / ChatGPT / Cursor / Copilot / Codex / MCPJam each carry an `mcpAppsCapabilities` matrix tuned to match their currently-advertised `HostCapabilities` blob byte-for-byte. Copilot now advertises the M365-published subset (no `serverResources` / `logging`, fullscreen-only display, no notification gates). Cursor's `listChanged: false` sub-field detail is preserved via a new preset-only `hostCapabilitiesAugment` (not user-editable).
- **Sparse user override layer**: `mcpProfile.apps.mcpAppsOverrides` accepts a sparse `McpAppsCapabilities` and merges over the host style preset via `mergeMcpAppsCapabilities` + `resolveEffectiveMcpAppsCapabilities`. `availableDisplayModes` is replaced (not unioned); empty arrays coerce to `["inline"]` (spec default).
- **Migration helper for legacy `hostCapabilitiesOverride`**: `hostCapabilitiesOverrideToMatrix(legacy)` converts the existing top-level `HostConfigInputV2.hostCapabilitiesOverride` raw blob into the sparse matrix shape. Legacy field stays readable for one release window; new writes prefer `mcpAppsOverrides`. Precedence rule: matrix override wins if both are present.
- **No behavior change for non-Copilot hosts**: `resolveEffectiveHostCapabilities` now folds the matrix when the new override is set, otherwise routes through the legacy strip-then-return path and the matrix-derived preset blob. Snapshot equality preserved for Claude / ChatGPT / Cursor / Codex / MCPJam (verified in updated `registry.test.ts` + `mcp-apps-renderer.test.tsx`).
- **Foundation only**: no notification-emission gates, no resource-meta gates, no annotation gates, no matrix UI in `AppsExtensionTab.tsx` yet. Those land in subsequent PRs (B/C/D in the foundation series). This PR is behavior-neutral aside from Copilot's now-correct advertisement and is safe to ship before the matrix editor is built.
