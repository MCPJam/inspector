## Overview

This changelog tracks all notable changes to MCPJam Inspector. We follow [Semantic Versioning](https://semver.org/) and keep our changelog in the spirit of [Keep a Changelog](https://keepachangelog.com/).

## Latest Releases

### Version 1.3.12

#### Added

- **MCP Tasks Support**
  - Full implementation of MCP Tasks specification (2025-11-25)
  - New Tasks tab for monitoring and managing background operations
  - Task-augmented tool execution with TTL configuration
  - Real-time progress tracking with determinate and indeterminate progress
  - Task status monitoring (working, input_required, completed, failed, cancelled)
  - Task cancellation support for compatible servers
  - Elicitation handling for tasks requiring user input
  - Local task tracking for servers without persistent task lists
  - Auto-refresh with server-suggested poll intervals
  - Task capabilities detection (tasks.requests.tools.call, tasks.list, tasks.cancel)

- **SDK Enhancements**
  - Upgraded `@modelcontextprotocol/sdk` to v1.25.0
  - Extended `MCPClientManager` with task management methods
  - Progress notification handling and storage
  - Task-aware elicitation support with relatedTaskId metadata

- **Server Infrastructure**
  - New `/api/mcp/tasks` endpoints for task operations
  - In-memory progress store with automatic cleanup
  - Global elicitation callback for task-related requests
  - Task result retrieval with proper metadata

#### Improved

- **Tool Execution**
  - Optional background task execution for long-running operations
  - Task requirement detection from tool metadata
  - Deep linking to Tasks tab after task creation

- **Progress Tracking**
  - Inline progress components in task list
  - Progress bar for determinate progress
  - Indeterminate progress animation for unknown totals
  - Elapsed time display for active tasks

### Version 1.1.0

#### Improved

- **Code Quality**
  - Improved code consistency and maintainability

### Recent Updates

#### Added

- **OAuth Protocol 2025-03-26 Support**
  - Added support for original MCP OAuth specification (2025-03-26)
  - Direct RFC8414 discovery from MCP server base URL with fallback endpoints
  - PKCE required for all clients
  - DCR and pre-registered client registration strategies

- **OAuth Debugger Enhancements**
  - Edit Config button to modify server settings without reconnecting
  - Copy logs feature with guide and raw formats for debugging
  - Deprecated transport detection with migration guidance for HTTP+SSE servers
  - RFC 8707 compliant resource URL canonicalization

- **GPT-5 Model Support**
  - Added support for GPT-5 model variants: `gpt-5`, `gpt-5-mini`, `gpt-5-nano`, `gpt-5-chat-latest`, `gpt-5-pro`, `gpt-5-codex`
  - Organization verification notice for GPT-5 access

#### Improved

- **OAuth Flow**
  - Refined diagram focusing and zoom behavior for better step visualization
  - Improved error handling for expected 4xx responses during transport detection
  - Enhanced header merging in OAuth state machines across all protocol versions

- **Error Handling**
  - Enhanced streaming error handling in chat interface
  - Errors now display as inline alerts instead of failing silently
  - Better error messages from AI providers
