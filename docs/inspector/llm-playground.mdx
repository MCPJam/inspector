The Chat tab lets you chat with your MCP server using MCPJam-provided free models or, if needed, your own provider. Test how different models interact with your tools, simulate agent behaviors, and render supported ChatGPT apps and MCP apps inline in the conversation.

## Key Features

- **MCPJam Free Models** - Start immediately with MCPJam-provided models without configuring your own provider
- **Inline Tool Results and Widgets** - See tool calls, results, and supported app UIs directly in the chat thread
- **Real-time Debugging** - Split-panel view with chat on the left and JSON-RPC logs on the right
- **Skills and Prompts** - Use `/` to inject skills and MCP prompts directly into chat
- **Elicitation Support** - Interactive prompts and forms from your MCP server
- **Agent Controls** - Tune system prompt, temperature, tool approval, context usage, and X-Ray from the chat toolbar

## Setup

Start with MCPJam free models:

1. **Open the Chat tab** - Connect your server and open Chat.
2. **Select a model** - Choose an MCPJam-provided model from the model dropdown.
3. **Start chatting** - Send a message and inspect tool calls, prompts, widgets, and logs inline.

## ChatGPT Apps and MCP Apps

Chat can render custom UI components from both [ChatGPT apps](https://developers.openai.com/apps-sdk) and [MCP apps](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/1865):

**ChatGPT Apps**
Tools with `openai/outputTemplate` metadata render custom HTML interfaces in an isolated iframe with access to the `window.openai` API.

**MCP Apps**
Tools with `ui.resourceUri` metadata render custom UI components inline in the chat.

Widgets can display interactive visualizations, trigger tool calls, send follow-up messages, and open external links.

## Chat Layout

The Chat tab uses a split-panel interface:

**Chat Panel (Left)**
- Send messages and view LLM responses
- See tool calls and results inline
- View rendered UI from supported ChatGPT apps and MCP apps
- Access skills and MCP prompts by typing `/`

**JSON-RPC Logger (Right)**
- Real-time view of MCP protocol messages
- Inspect requests and responses between the inspector and your servers
- Debug tool invocations and error responses

Resize panels by dragging the divider to focus on chat or debugging as needed.

## Chat Controls

Use the chat input toolbar to customize and inspect model behavior:

- **Context Usage** - Open the context meter to see how much of the selected model's context window is being used
- **System Prompt** - Set instructions and context for the LLM
- **Temperature** - Control randomness (0 = deterministic, higher = more creative)
- **Tool Approval** - Require manual approval before the model runs a tool
- **X-Ray** - Open a read-only JSON view of what Chat sends to the model

<Info>
  Higher temperature settings may cause more hallucinations with MCP tool interactions. Start with default values.
</Info>

Open the context meter in the toolbar to inspect the token breakdown for the current conversation, including input, output, system prompt, and MCP tools usage.

When X-Ray is open, the chat thread is replaced with a raw payload view so you can inspect the system prompt, tools, and message history being sent to the model.

## Skills and Prompts

Type `/` in chat to open the picker for skills and MCP prompts. Skills help the model use tools and complete workflows more effectively, and prompts let you insert reusable MCP prompt templates directly into the conversation.

When you select a prompt, it appears as an expandable card showing:

- Server name and description
- Required and optional arguments
- Preview of the prompt content

Selected skills and prompts appear above the input before you send your message, so you can review or remove them first.

### Optional: Bring Your Own Provider

If you need a specific external model, go to **Settings** and configure your own provider. MCPJam supports built-in configuration for OpenAI, Anthropic, Google, DeepSeek, Mistral, xAI, Ollama, OpenRouter, and Azure OpenAI.
