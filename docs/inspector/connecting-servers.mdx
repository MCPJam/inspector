MCPJam can connect to both local and remote MCP servers.

The simple version:

- `STDIO` means MCPJam starts a program on your machine
- `SSE` or `Streamable HTTP` means MCPJam connects to a server URL
- you can stay connected to more than one server at the same time

## Manage connections

Go to the "MCP Servers" tab to add, remove, or inspect servers.

- Toggle MCP servers to enable or temporarily deactivate a server.
- Hit the 3 dots to reconnect, edit, or disconnect a server.
- Click "View server info" to open a detailed modal showing server metadata, protocol version, capabilities, and instructions.

### Server info modal

The server info modal shows:

- **Server metadata**: Name, title, version, protocol version, and transport type
- **Capabilities**: Tools, prompts, and resources supported by the server
- **Instructions**: Server-provided instructions for AI assistants
- **Raw capabilities**: Full JSON of server and client capabilities (with copy button)
- **OpenAI Apps SDK**: For servers using the OpenAI Apps SDK, view widget metadata in a separate tab

### What each server card shows

Each connected server card displays:

- **Server icon and version**: If provided by the server, the icon and version number appear in the card header
- **Connection status**: Real-time indicator showing connected, connecting, or failed status with retry count
- **Transport type**: Shows whether the server uses STDIO, SSE, or Streamable HTTP

Click "Show details" to expand additional server information:

- **Server capabilities**: Lists available features (Tools, Prompts, Resources)
- **MCP protocol version**: The protocol version the server implements
- **Instructions**: Server-provided instructions for AI assistants
- **Documentation link**: Direct link to the server's documentation if available

### OpenAI app metadata

Some servers also expose OpenAI-compatible tool metadata. Those show an OpenAI logo and two tabs:

- **Info tab**: Standard server information and capabilities
- **OpenAI Tool Metadata tab**: Detailed metadata for each tool including read/write permissions and widget configuration

<Note>
  Using the [hosted app](https://app.mcpjam.com)? Only SSE and Streamable HTTP connections are available. STDIO and HTTP (non-HTTPS) connections require the [local inspector](/installation). See [Hosted App](/hosted/overview) for details.
</Note>

## Local servers with STDIO

Use STDIO when the server lives on your machine and should be started with a command.

Example:

```bash
node /path/to/your/server.js
```

<Info>
  Use absolute paths (for example,
  `/Users/yourname/project/server.py`) so MCPJam can always find your files.
</Info>

### Environment variables

Need to pass a token or config value to a local server? Use "Add Variable" in the Environment Variables section.

## Remote servers with SSE or Streamable HTTP

Use these when the server is already running somewhere else and gives you a URL.

Choose `SSE/HTTP` in the transport dropdown, then paste the URL.

Common pattern:

- SSE URLs often end in `/sse`
- Streamable HTTP URLs often end in `/mcp`

### OAuth 2

If the remote server needs auth, MCPJam supports the MCP OAuth flow and Dynamic Client Registration (DCR).

You can choose:

1. Choose `No Authentication`
2. Choose `Bearer Token`. Use this if you already have an API token and don't need to go through DCR.
3. Choose `OAuth 2.0` if you want MCPJam to walk through the auth flow and fetch a bearer token for you. You can also set custom scopes or credentials.

## Tunneling

Tunnels let you give a local server a public HTTPS URL for temporary remote access.

### Create a tunnel

1. Go to the **MCP Servers** tab
2. Click **Create Tunnel** in the header
3. MCPJam copies the tunnel URL to your clipboard

Each connected server gets its own tunnel URL. Click **Copy Tunnel Url** on a server card to copy that server's endpoint.

### Close a tunnel

Click **Close Tunnel** to stop remote access right away.

<Warning>
  Tunnel URLs are publicly accessible without authentication. Only create
  tunnels when you need remote access, and close them when done.
</Warning>
