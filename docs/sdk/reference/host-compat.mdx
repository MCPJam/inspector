The host compatibility SDK lets you ask "where can I ship this?" — evaluating your MCP Apps server's tools against the 10 real-world hosts that developers target (Claude, ChatGPT, Cursor, Copilot, Goose, Mistral, Codex, n8n, Perplexity, and Cline).

## Import

```typescript
import {
  evaluateMarketHosts,
  buildMarketHostProfiles,
  MCP_APPS_FULL,
  MCP_APPS_CHATGPT,
  MCP_APPS_MISTRAL,
  MCP_APPS_CURSOR,
  MCP_APPS_GOOSE,
  MCP_APPS_COPILOT,
  MCP_APPS_NO_CLAIMS,
} from "@mcpjam/sdk/host-compat";
```

## Basic usage

Pass your server's tools and metadata to `evaluateMarketHosts`. Each report in the result tells you whether the tool works, degrades, or is unsupported on that host.

```typescript
import { evaluateMarketHosts } from "@mcpjam/sdk/host-compat";

const result = evaluateMarketHosts({
  tools: [{ name: "my_widget" }],
  toolsMetadata: {
    my_widget: {
      ui: { resourceUri: "ui://my-widget" },
    },
  },
});

for (const report of result.reports) {
  console.log(`${report.hostId}: ${report.verdict}`);
  // claude: works
  // chatgpt: works
  // codex: degraded   (headless — no widget rendering surface)
  // ...
}
```

### Passing widget usage

If your widget uses specific host APIs (such as `ui/message`), pass `widgetUsage` so the evaluator can check whether each host supports those APIs:

```typescript
const result = evaluateMarketHosts(
  {
    tools: [{ name: "my_widget" }],
    toolsMetadata: {
      my_widget: { ui: { resourceUri: "ui://my-widget" } },
    },
  },
  {
    widgetUsage: {
      message: ["my_widget"],   // my_widget calls ui/message
    },
  },
);

// cursor: degraded  (Cursor lacks ui/message)
// goose: degraded   (Goose lacks ui/message)
// claude: works
```

## `evaluateMarketHosts(toolsData, options?)`

Evaluates a server against the default 10-host market catalog. A thin wrapper around `evaluateAllHosts(..., buildMarketHostProfiles())`.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `toolsData` | `HostCompatToolsInput \| null \| undefined` | No | Tools and metadata from your server. Pass `null` or `undefined` to evaluate with no tools. |
| `options` | `EvaluateAllHostsOptions` | No | Optional evaluation hints, including `widgetUsage`. |

Returns `HostCompatEvaluation`:

| Property | Type | Description |
|----------|------|-------------|
| `reports` | `HostCompatReport[]` | One report per host in the catalog. |

Each `HostCompatReport` includes:

| Property | Type | Description |
|----------|------|-------------|
| `hostId` | `string` | Host identifier (e.g. `"claude"`, `"chatgpt"`, `"codex"`). |
| `verdict` | `"works" \| "degraded" \| "unsupported"` | Overall compatibility verdict for this host. |

## `buildMarketHostProfiles()`

Returns the 10 market-host profiles as `HostCompatProfile[]`. Each call returns fresh copies — mutating the returned array or its profiles does not affect subsequent calls.

```typescript
import { buildMarketHostProfiles } from "@mcpjam/sdk/host-compat";

const profiles = buildMarketHostProfiles();

// Inspect a specific host
const goose = profiles.find((p) => p.id === "goose");
console.log(goose?.rendersOpenAiApps);          // false
console.log(goose?.capabilities?.message);      // false
console.log(goose?.supportedProtocolVersions);  // ["2025-03-26"]
```

Each `HostCompatProfile` includes:

| Property | Type | Description |
|----------|------|-------------|
| `id` | `string` | Host identifier. |
| `label` | `string` | Display name (e.g. `"Claude"`, `"ChatGPT"`). |
| `provenance` | `CompatProvenance` | Data source: `"probe"`, `"vendor-doc"`, or `"assumed"`. |
| `rendersMcpApps` | `boolean` | Whether the host renders MCP Apps widgets. `false` for headless hosts (Codex, n8n, Perplexity, Cline). |
| `rendersOpenAiApps` | `boolean` | Whether the host injects the `window.openai` bridge (ChatGPT and Copilot). |
| `supportedProtocolVersions` | `string[] \| undefined` | MCP base-protocol versions the host advertises. `undefined` when the host doesn't pin a version. |
| `capabilities` | `McpAppsCapabilities \| undefined` | Per-host MCP Apps capability matrix. `undefined` for headless hosts. |

## Capability matrix constants

The per-host MCP Apps capability matrices are exported as frozen constants. Use them when building custom host profiles or comparing capability sets.

| Constant | Host |
|----------|------|
| `MCP_APPS_FULL` | Claude — every capability on. |
| `MCP_APPS_CHATGPT` | ChatGPT — full minus `serverResources` and `logging`. |
| `MCP_APPS_MISTRAL` | Mistral — probe-captured; no pip, download, or teardown. |
| `MCP_APPS_GOOSE` | Goose Desktop — only `openLinks` and `toolInfo` advertised. |
| `MCP_APPS_CURSOR` | Cursor — full minus `updateModelContext` and `message`. |
| `MCP_APPS_COPILOT` | Microsoft 365 Copilot — published component-bridge table. |
| `MCP_APPS_NO_CLAIMS` | Spec-default fallback — every capability off. |

All constants are deeply frozen. Attempting to mutate them throws in strict mode.

```typescript
import { MCP_APPS_FULL, MCP_APPS_CURSOR } from "@mcpjam/sdk/host-compat";

// Check whether a specific capability differs between hosts
console.log(MCP_APPS_FULL.message);    // true
console.log(MCP_APPS_CURSOR.message);  // false
```

## `EvaluateAllHostsOptions`

| Property | Type | Description |
|----------|------|-------------|
| `widgetUsage` | `Record<string, string[]>` | Maps API names (e.g. `"message"`, `"serverTools"`) to the tool names that use them. Used to check whether each host supports the APIs your widgets call. |

## Notes

- MCPJam itself is not included in the market catalog — it is compatible by construction.
- Headless hosts (Codex, n8n, Perplexity, Cline) always produce a `"degraded"` verdict for MCP Apps widgets because they have no rendering surface.
- `provenance: "assumed"` means the values are best-effort presets, not captured from a live host or vendor documentation.
- The catalog is built once and cached internally. `buildMarketHostProfiles()` returns deep copies on each call so callers can sort or modify the array without affecting later evaluations.
