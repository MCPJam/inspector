## Quick start

Run this in your terminal:

```bash
npx @mcpjam/inspector@latest
```

When the install finishes, open the `localhost` link it prints.

<Note>
  Don't have Node.js installed? Download our desktop app for [Mac](https://github.com/MCPJam/inspector/releases/latest/download/MCPJam.Inspector.dmg) or [Windows](https://github.com/MCPJam/inspector/releases/latest/download/MCPJam-Inspector-Setup.exe).
</Note>

## Docker

Want to run MCPJam with Docker instead? Use:

```bash
docker run -p 127.0.0.1:6274:6274 mcpjam/mcp-inspector
```

The application will be available at **`http://127.0.0.1:6274`**.

<Warning>
**Security:** Always use `-p 127.0.0.1:6274:6274` (not `-p 6274:6274`) to ensure the inspector is only accessible from your local machine.
</Warning>

### Docker with Arguments

You can also pass startup arguments into the container:

```bash
# Launch with STDIO server
docker run -p 127.0.0.1:6274:6274 mcpjam/mcp-inspector -- \
  npx -y @modelcontextprotocol/server-everything

# Launch with HTTP server
docker run --rm -p 127.0.0.1:6274:6274 mcpjam/mcp-inspector -- \
  --url http://host.docker.internal:8080/mcp \
  --name "HTTP Server" \
  --tab tools
```

For more details, see [Launch from Code](https://docs.mcpjam.com/inspector/launch-from-code).

<Warning>
**Important for macOS/Windows users:**

- Access the app via `http://127.0.0.1:6274` (not `localhost`)
- When connecting to MCP servers on your host machine, use `http://host.docker.internal:PORT` instead of `http://127.0.0.1:PORT`

Example:

```bash
# Your MCP server runs on host at: http://127.0.0.1:8080/mcp
# In Docker, configure it as: http://host.docker.internal:8080/mcp
```

</Warning>

## Start with a STDIO server

This starts MCPJam and connects it to a local STDIO server right away.

<Tabs>
  <Tab title="Python Server">
    ```bash
    # Starts MCPJam inspector with a FastMCP server
    npx @mcpjam/inspector@latest uv run fastmcp run /path/to/your/server.py
    ```
  </Tab>
  
  <Tab title="Node.js Server">
    ```bash
    # NPX package
    npx @mcpjam/inspector@latest npx -y your-mcp-package

    # Direct Node.js execution
    npx @mcpjam/inspector@latest node /path/to/your/server.js
    ```

  </Tab>
</Tabs>

<Info>
  **Use Absolute Paths**: Always use absolute file paths (e.g.,
  `/Users/yourname/project/server.py`) to avoid path resolution issues.
</Info>

## Start with a custom port

The default port is `6274`. Change it like this:

```bash
npx @mcpjam/inspector@latest --port 4000
```

## Start with Ollama

This starts MCPJam and launches an Ollama model with `ollama serve <model>`. Make sure Ollama is installed first.

```bash
npx @mcpjam/inspector@latest --ollama llama3.2
```

## Start with a configuration file

Use a config file when you want MCPJam to load multiple servers at once:

```bash
# Using npx directly
npx @mcpjam/inspector@latest --config path/to/config.json --server myserver

# If running from the repository
npm run build
npm start -- --config path/to/config.json --server myserver
```

<Info>
  **Note**: When using `npm start`, you need the `--` separator to pass
  arguments to the underlying script.
</Info>

Example config:

```json
{
  "mcpServers": {
    "everything": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-everything"],
      "env": {
        "HELLO": "Hello MCP!"
      }
    },
    "weather-server": {
      "command": "python",
      "args": ["weather_server.py"],
      "env": {
        "WEATHER_API_KEY": "your-api-key"
      }
    }
  }
}
```

## Start with verbose logging

Use verbose logging when you need to debug connection problems.

```bash
npx @mcpjam/inspector@latest --verbose
```

Or use the short form:

```bash
npx @mcpjam/inspector@latest -v
```

<Info>
  Verbose mode logs the inspector's HTTP requests so you can see what it is
  sending and receiving.
</Info>
